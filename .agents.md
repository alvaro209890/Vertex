# Vertex

Standalone Vertex CLI app backed by a local DeepSeek proxy. The user-facing
product is `vertex`; `vertex-proxy` is an internal/server mode used by the app
and by advanced users who need only the Anthropic-compatible HTTP server.

Vertex was created by Alvaro Emanuel Alves Araujo. It is not an Anthropic
product. The bundled assistant must identify as Vertex and explain that it runs
DeepSeek models through the local proxy.

## Quick Start

```bash
# Dependencies: uv, Python 3.12+, Node.js 20+
uv sync
vertex
```

On first run, `vertex` asks for `DEEPSEEK_API_KEY` and writes
`~/.config/vertex/.env`. `vertex-init` can pre-create that file, but it is not
required.

## Architecture

- **vertex** — main app command. Starts the local proxy if needed and then opens
  the bundled CLI runtime.
- **vertex-proxy** — proxy-only/server mode. This is a support command, not the
  primary product entrypoint.
- **cli/entrypoints.py** — Python wrapper for install-time commands, first-run
  setup, DeepSeek API-key login/logout, proxy startup, and vendored CLI launch.
- **vendor/vertex-cli/bin/vertex** — Node launcher for the bundled runtime.
- **vendor/vertex-cli/dist/cli.mjs** — vendored, built CLI bundle. Do not edit
  this file by hand without also updating the patch in `patches/`.
- **patches/vertex-cli-disable-anthropic-login.patch** — canonical patch that
  disables Anthropic account login paths and keeps Vertex API-key-only.
- **scripts/apply-vertex-cli-patches.sh** — reapplies the vendored CLI patch
  after refreshing `vendor/vertex-cli`.
- **server.py** — ASGI entrypoint (uvicorn)
- **api/app.py** — FastAPI app factory with lifespan, error handlers
- **api/routes.py** — Route handlers (`/v1/messages`, `/v1/models`, `/health`, etc.)
- **config/settings.py** — Pydantic settings from `.env` / environment
- **config/provider_catalog.py** — DeepSeek provider descriptor
- **providers/registry.py** — Provider factory + runtime cache
- **providers/deepseek/** — DeepSeek Anthropic-native transport
- **core/anthropic/** — Shared Anthropic protocol utilities
- **cli/** — CLI session management, entrypoints
- **messaging/** — Optional Discord/Telegram bot adapters

## Product Boundary

`vertex` is the app. It owns first-run configuration, starts `vertex-proxy`, and
launches the bundled CLI. The CLI talks to the local proxy through
`ANTHROPIC_BASE_URL=http://127.0.0.1:8083`; users should not need to know or run
the proxy for normal use.

`vertex-proxy` remains available for smoke tests, external Anthropic-compatible
clients, and advanced integrations. Treat it as infrastructure behind the app.

Anthropic account login is intentionally disabled. The only supported user login
state is the DeepSeek API key stored in `~/.config/vertex/.env`.

## Vendored CLI Patch Workflow

The vendored CLI is currently shipped as a built bundle at
`vendor/vertex-cli/dist/cli.mjs`. Because that file is large and generated,
runtime changes must be tracked as a reusable patch:

1. Update or replace `vendor/vertex-cli`.
2. Run `scripts/apply-vertex-cli-patches.sh`.
3. If the patch fails, refresh `patches/vertex-cli-disable-anthropic-login.patch`
   against the new bundle.
4. Run `node --check vendor/vertex-cli/dist/cli.mjs`.
5. Run the Python checks and CLI tests.

Do not reintroduce Anthropic OAuth, Claude subscription, or Anthropic Console
login as a supported Vertex login path. If the upstream CLI changes auth code,
preserve the Vertex invariant: DeepSeek API key only.

## Models

| Model ID | Description |
|---|---|
| `deepseek/deepseek-v4-pro` | Full reasoning (thinking mode) |
| `deepseek/deepseek-chat` | Fast responses (non-thinking) |
| `deepseek/deepseek-chat` | Legacy (deprecated 2026/07/24) |
| `deepseek/deepseek-reasoner` | Legacy (deprecated 2026/07/24) |

## Configuration (`.env`)

| Variable | Description |
|---|---|
| `DEEPSEEK_API_KEY` | DeepSeek API key (required) |
| `MODEL` | Default model |
| `MODEL_OPUS` | Override for Opus-tier requests |
| `MODEL_SONNET` | Override for Sonnet-tier requests |
| `MODEL_HAIKU` | Override for Haiku-tier requests |
| `ANTHROPIC_AUTH_TOKEN` | Proxy auth token |
| `MESSAGING_PLATFORM` | "telegram", "discord", or "none" |
| `ENABLE_MODEL_THINKING` | Enable thinking/reasoning blocks |
| `PORT` | Server port (default 8083) |

## API Endpoints

- `POST /v1/messages` — Stream a message (Anthropic Messages API)
- `POST /v1/messages/count_tokens` — Count tokens
- `GET /v1/models` — List available models
- `GET /health` — Health check

## Clients

- **External Anthropic-compatible CLI**: `ANTHROPIC_AUTH_TOKEN="freecc" ANTHROPIC_BASE_URL="http://localhost:8083" claude`
- **VS Code**: Set `claudeCode.environmentVariables` with `ANTHROPIC_BASE_URL` and `ANTHROPIC_AUTH_TOKEN`
- **JetBrains**: Edit `~/.jetbrains/acp.json` or Windows equivalent

## Commands

- `vertex` — Main Vertex app. Runs first-run setup, starts the proxy, launches the CLI.
- `vertex /logout` — Replace the DeepSeek API key.
- `vertex auth login` — Replace the DeepSeek API key.
- `vertex auth status` — Report DeepSeek API-key status.
- `vertex-init` — Optional config scaffold at `~/.config/vertex/.env`
- `vertex-proxy` — Proxy-only mode for advanced usage and smoke tests.
- `claude-pick` — Interactive model picker (requires `fzf`)
